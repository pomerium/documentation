# Default Upstream Timeout

<Tabs>
<TabItem value="Core" label="Core">

---

**Config file keys:** `default_upstream_timeout` <br/>
**Environment variables:** `DEFAULT_UPSTREAM_TIMEOUT` <br/>
**Type:** `string` <br/>
**Default:** `30s`

**Example:**
```yaml
default_upstream_timeout: 30s

DEFAULT_UPSTREAM_TIMEOUT: 1h45m
```

</TabItem>
<TabItem value="Enterprise" label="Enterprise">

---

**Config file keys:** `default_upstream_timeout` <br/>
**Environment variables:** `DEFAULT_UPSTREAM_TIMEOUT` <br/>
**Type:** `string` <br/>
**Default:** `30s`

![Enterprise Console with default timeout setting](./img/timeouts/timeouts-default.png)

</TabItem>
<TabItem value="Kubernetes" label="Kubernetes">

---

Kubernetes does not support `default_upstream_timeout`

</TabItem>
</Tabs>

---

**Default Upstream Timeout** is the default timeout applied to a proxied route when no `timeout` key is specified by the policy.

See the following docs for more information:
- [Global Timeouts](/docs/reference/global-timeouts)
- [GO Duration](https://pkg.go.dev/time#Duration)
