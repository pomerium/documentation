# Secure Cookie

<Tabs>
<TabItem value="Core" label="Core">

---

**Config file keys:** `cookie_secure` <br/>
**Environment variables:** `COOKIE_SECURE` <br/>
**Type:** `boolean` <br/>
**Default:** `true`


**Example:**
```yaml
cookie_secure: false

COOKIE_SECURE=false
```

</TabItem>
<TabItem value="Enterprise" label="Enterprise">

---

**Config file keys:** `cookie_secure` <br/>
**Environment variables:** `COOKIE_SECURE` <br/>
**Type:** `boolean` <br/>
**Default:** `true`

In the Enterprise Console, configure `cookie_secure` with the **HTTPS only** toggle button. The button has three states:
- **Unset** ("-") uses the value in your `config.yaml` file
- **Checkmark** sets `cookie_secure` to `true`
- **Empty** sets `cookie_secure` to `false`

</TabItem>
<TabItem value="Kubernetes" label="Kubernetes">

---

**Name:** `cookie.secure`
**Type:** `string`
**Default:** `true`

**Example:**
```yaml
cookie.secure: false
```

See Kubernetes [Cookie reference](/docs/deploying/k8s/reference#cookie) for more information.

</TabItem>
</Tabs>

---

If true, **Cookie Secure** instructs browsers to only send user session cookies over HTTPS.

:::warning

Setting this to `false` may result in session cookies being sent in clear text.

:::
