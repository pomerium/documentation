# Authenticate Internal Service URL

<Tabs>
<TabItem label="Config file key" value="Config file key">

---

**Name:** `authenticate_internal_service_url` <br/>
**Type:** `URL` <br/>
**Optional**

**Example:**

```yaml
authenticate_internal_service_url: https://authenticate.internal
```

</TabItem>
<TabItem label="Environment variable" value="Environment variable">

---

**Name:** `AUTHENTICATE_INTERNAL_SERVICE_URL` <br/>
**Type:** `URL` <br/>
**Optional**

**Example:**

```yaml
AUTHENTICATE_INTERNAL_SERVICE_URL=https://authenticate.internal
```

</TabItem>
<TabItem label="Kubernetes" value="Kubernetes">

---

Kubernetes does not support `authenticate_internal_service_url`

</TabItem>
</Tabs>

---

**Authenticate Internal Service URL** provides a separate TLS Certificate and Hostname to the Authenticate Service when it determines which TLS Certificate and Hostname to use when verifying a downstream server.

If included, **Authenticate Internal Service URL** overrides the TLS Certificate and Hostname provided by [**Authenticate Service URL**](/docs/reference/authenticate-service-url).
