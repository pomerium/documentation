# Regex Rewrite

## Summary

If **Regex Rewrite** is set, the URL path will be rewritten according to the pattern and substitution, similar to [`prefix_rewrite`](/docs/reference/routes/prefix-rewrite).

## How to configure

| **Type** | **Usage**    |
| :------- | :----------- |
| `string` | **optional** |

<Tabs>
<TabItem value="Core" label="Core">

| **YAML**/**JSON** setting    |
| :--------------------------- |
| `regex_rewrite_pattern`      |
| `regex_rewrite_substitution` |

</TabItem>
<TabItem value="Enterprise" label="Enterprise">

Set **Regex Rewrite** pattern and substitution in the Console:

![Set regex rewrite pattern and substitution in the console](../img/routes/regex/regex-rewrites.png)

</TabItem>
<TabItem value="Kubernetes" label="Kubernetes">

Kubernetes does not support **Regex Rewrite**

</TabItem>
</Tabs>

### Examples

```yaml
regex_rewrite_pattern: '^/service/([^/]+)(/.*)$'
regex_rewrite_substitution: "\\2/instance/\\1"
```
