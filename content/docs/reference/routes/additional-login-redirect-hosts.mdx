## Summary

Sometimes a single-page application (SPA) may require the use of multiple domains. In this case, sometimes CORS restrictions can prevent Pomerium login from working seamlessly.

The **Additional Login Redirect Hosts** option configures a Pomerium route so that when a user logs in at that route domain, they will automatically receive a Pomerium session cookie for several additional domains.

Up to five additional domains can be specified on a single Pomerium route.

:::info

Currently this setting cannot be used in combination with the [Hosted Authenticate Service](/docs/capabilities/authentication#hosted-authenticate-service).

:::

## How to configure

<Tabs>
<TabItem value="Core" label="Core">

| Config file key | Environment variable | Type | Default |
| :-- | :-- | :-- | :-- |
| `depends_on` | `DEPENDS_ON` | array of strings | _none_ |

### Examples

```yaml
routes:
 - from: https://my-app.example.com
   to: https://my-app-backend.internal
   depends_on:
    - api.my-app.example.com
    - assets.my-app.example.com
```

</TabItem>
<TabItem value="Enterprise" label="Enterprise">

**Additional Login Redirect Hosts** is listed under the "General" section of the route settings.

</TabItem>
<TabItem value="Kubernetes" label="Kubernetes">

| [Annotation name](/docs/deploy/k8s/ingress#set-ingress-annotations) | Type | Usage | Default |
| :-- | :-- | :-- | :-- |
| `depends_on` | array of strings | **optional** | _none_ |

### Examples

```yaml
ingress.pomerium.io/depends_on: |
 - api.my-app.example.com
 - assets.my-app.example.com
```

```yaml
ingress.pomerium.io/depends_on: '["api.my-app.example.com", "assets.my-app.example.com"]'
```

See [Kubernetes - Ingress Configuration](/docs/deploy/k8s/ingress) for more information.

</TabItem>
