# JWT Allowed Audiences

## Summary

**JWT Allowed Audiences** is the allowlist of audiences Pomerium accepts when a route uses the `jwt` [bearer token format](/docs/reference/bearer-token-format). A token is accepted only if its `aud` claim includes at least one of these values.

This check is **fail-closed**: if the effective allowlist is empty, every token is rejected. There is no "accept any audience" mode. A route using the `jwt` format requires a non-empty effective allowlist, or Pomerium refuses to start.

Trusted issuers are declared separately, via [`jwt_allowed_issuers`](/docs/reference/jwt-allowed-issuers); authorization on the verified claims is left to your [policy](/docs/internals/ppl) (`claim/…`).

This option sets the cluster-wide default. It can also be configured at the route level, where it replaces (does not merge with) the global value for that route. See [Machine-to-Machine Access with Bearer Tokens](/docs/capabilities/bearer-token-access) for an overview and examples.

### Examples

```yaml

  - pomerium.example.com
```
